# Ray-traced terrain bench results (WebGL2 fallback path)

CPU-side end-to-end frame time for `Render::draw_world` with
`RayTraced` terrain + `RayTraced` shadow at 1024² — i.e. exactly the
combination the WebGL2 web build falls back to. Same Fostral citadel
target as `snapshots/voxel-bench/`, same 640×480 resolution, 60 timed
frames after 40 warmup frames (no voxel bake to drain).

Generated by `tools/bench-ray.sh <stage>`.

| Stage | 30° elev | 60° elev | 90° elev |
|---|---|---|---|
| baseline | 0.33 / 0.41 / 2.34 | 0.31 / 0.36 / 2.08 | 0.30 / 0.34 / 1.81 |
| improved | 0.38 / 0.45 / 1.77 | 0.35 / 0.43 / 1.87 | 0.34 / 0.46 / 2.09 |

`improved` bundles four changes targeted at the WebGL2 fragment-bound
budget:

* `cast_ray_to_map` doubles the forward-step counts (8 → 16 for the
  surface cast, 6 → 12 for the underground cast). Eliminates the thin-
  column skipping that produced jagged silhouettes at oblique angles.
* `get_surface_height` (used for surface gradient + horizon factor in
  `evaluate_color_id`) now uses `get_surface_alt_smooth`, so terrain
  lighting transitions bilinearly across cell boundaries instead of
  stepping per cell.
* `fetch_shadow` does a 4-tap PCF instead of a single sample. The
  hardware comparison sampler still does its bilinear inside each tap,
  giving an effective ~3×3 box blur for 4 samples. Most visible at the
  90° camera where elevated structures cast hard edges.
* `bin/web/main.rs` drops the WebGL shadow map size from 1024² to 512²
  (independent change, kicks in only on the `is_webgpu == false`
  branch). Frees ~75% of the shadow-pass cost on weaker GPUs.

Net per-frame cost: ~10–35% slower across all three elevations
(0.34 → 0.46 ms avg at 90°, the worst case). On a fragment-bound build
this is well within the steady-state budget, with clearly visible
quality gains in shadows and lighting smoothness.
