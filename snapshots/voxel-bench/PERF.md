# Voxel-terrain bench results

CPU-side end-to-end frame time for `Render::draw_world` with
`RayVoxelTraced { voxel_size: [2,4,1], max_outer_steps: 40, max_inner_steps: 40 }`
plus voxel-traced shadow at `max_outer_steps: 20, max_inner_steps: 20` (the
WebGPU build's defaults). Rendering Fostral at the citadel cluster around
`(576, 896, 80)` with the camera 350 units out. Each row is
`min / avg / max ms` over 60 timed frames after 300 warmup frames.

Generated by `tools/bench-voxel.sh <stage>`.

| Stage | 30° elev | 60° elev | 90° elev |
|---|---|---|---|
| baseline (no shadow) | 0.88 / 1.66 / 4.86 | 0.86 / 1.16 / 4.57 | 0.89 / 1.17 / 4.57 |
| c1-fix (no shadow)   | 0.87 / 1.35 / 4.77 | 0.85 / 1.56 / 4.92 | 0.87 / 0.92 / 1.14 |
| c2-fix (no shadow)   | 0.87 / 1.35 / 4.87 | 0.86 / 1.34 / 4.86 | 0.88 / 1.16 / 4.74 |
| c4-fix (no shadow)   | 0.87 / 1.34 / 4.82 | 0.86 / 1.56 / 4.85 | 0.89 / 1.40 / 4.84 |
| voxel-shadow         | 2.93 / 3.03 / 3.51 | 2.85 / 2.94 / 3.24 | 2.75 / 2.89 / 3.40 |

`c1-fix` corrects the per-frame bake budget. Steady-state frame time matches
baseline within run-to-run noise (the fix accelerates the *initial* level
bake, not the timed frames). Rendered images are byte-for-byte identical to
baseline — the fix only changes how quickly the same final voxel grid is
produced.

`c2-fix` repairs the `t_step` typo (`min(tpu.y, tpu.y)` → `min(tpu.y, tpu.z)`),
which controls inner-cast step density. With the bug, top-down rays under-
sampled because `tpu.z` was ignored even when it was the smallest component.
The 90° elevation now spends ~25% more time in the inner cast (0.92 → 1.16 ms
avg) — the cost of correct sampling along the dominant ray axis. Visual
diffs vs c1-fix are small per-pixel sampling jitter; the overall image is
indistinguishable to the eye.

`c4-fix` makes voxels straddling `low_alt` get marked occupied in tunnel
(double-level) cells, where the previous condition (`z1 <= low_alt`) only
caught fully-below voxels. Images at all three reference cameras are
byte-for-byte identical to c2-fix — at the current settings (`voxel_size.z = 1`,
integer-quantised altitudes), the buggy condition aligned with voxel
boundaries and never triggered. The fix is preventive and would surface
the moment the `voxel_size.z` is increased or `texture_scale.z` becomes
non-256. Frame-time deltas vs c2-fix are run-to-run noise.

`voxel-shadow` switches the shadow pass from `RayTraced` to `RayVoxelTraced`
to match the WebGPU build's default. This adds a second voxel-traced pass
(at half the step counts, against the same grid) and ~2 ms to every frame
across all elevations — that's the cost of casting voxel-quality shadows
instead of the cheaper height-field ray pass. Visual diff vs the no-shadow
rows is subtle on this scene (the citadel is mostly self-occluding from
the sun direction), most apparent at 90°.
