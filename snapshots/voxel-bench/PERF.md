# Voxel-terrain bench results

CPU-side end-to-end frame time for `Render::draw_world` with
`RayVoxelTraced { voxel_size: [2,4,1], max_outer_steps: 40, max_inner_steps: 40 }`,
rendering Fostral at the citadel cluster around `(576, 896, 80)` with the camera
350 units out. Each row is `min / avg / max ms` over 60 timed frames after
300 warmup frames.

Generated by `tools/bench-voxel.sh <stage>`.

| Stage | 30° elev | 60° elev | 90° elev |
|---|---|---|---|
| baseline | 0.88 / 1.66 / 4.86 | 0.86 / 1.16 / 4.57 | 0.89 / 1.17 / 4.57 |
| c1-fix   | 0.87 / 1.35 / 4.77 | 0.85 / 1.56 / 4.92 | 0.87 / 0.92 / 1.14 |

`c1-fix` corrects the per-frame bake budget. Steady-state frame time matches
baseline within run-to-run noise (the fix accelerates the *initial* level
bake, not the timed frames). Rendered images are byte-for-byte identical to
baseline — the fix only changes how quickly the same final voxel grid is
produced.
